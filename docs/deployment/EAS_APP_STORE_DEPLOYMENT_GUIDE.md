## What Goes Into EAS

### EAS Deployment Scope

**‚úÖ What EAS Builds and Deploys:**
- **Frontend React Native app** (`NestSync-frontend/`)
- iOS app binary (.ipa file for App Store)
- Android app binary (.aab file for Google Play)
- Web export (optional, for browser deployment)

**‚ùå What EAS Does NOT Handle:**
- **Backend FastAPI server** (already deployed to Railway)
- **Database** (already on Supabase)
- **API infrastructure** (already containerized and running)

## Phase 3: Build Configuration

### Step 3.1: iOS Code Signing Setup

**Prerequisites:**
- Active Apple Developer account ($99/year)
- Access to App Store Connect
- Mac computer (for local certificate management) OR EAS managed credentials (recommended)

**Option A: EAS Managed Credentials (Recommended)**

EAS will automatically generate and manage certificates:

```bash
cd /home/user/NestSyncV1.2/NestSync-frontend

# EAS will prompt for Apple ID credentials during first build
eas build --platform ios --profile production
```

**Option B: Manual Certificate Management**

1. **Create App ID** in Apple Developer Portal:
   - Log in to https://developer.apple.com/account
   - Navigate to Certificates, Identifiers & Profiles
   - Create new App ID with Bundle ID: `ca.nestsync.mobile`
   - Enable capabilities: Push Notifications, Associated Domains

2. **Generate Distribution Certificate**:
   - Create new iOS Distribution certificate
   - Download and install in Keychain (Mac only)

3. **Create Provisioning Profile**:
   - Create App Store provisioning profile
   - Link to App ID and distribution certificate
   - Download and configure in EAS

4. **Upload to EAS**:
```bash
eas credentials --platform ios
# Follow prompts to upload certificates and provisioning profiles
```

### Step 3.2: Android Code Signing Setup

**Option A: EAS Managed Keystore (Recommended)**

EAS will generate and securely store your keystore:

```bash
cd /home/user/NestSyncV1.2/NestSync-frontend

# EAS generates keystore automatically during first build
eas build --platform android --profile production
```

**Option B: Manual Keystore Management**

1. **Generate Keystore**:
```bash
keytool -genkeypair -v -storetype PKCS12 -keystore nestsync.keystore \
  -alias nestsync-key -keyalg RSA -keysize 2048 -validity 10000
```

2. **Upload to EAS**:
```bash
eas credentials --platform android
# Choose "Use an existing Keystore" and upload nestsync.keystore
```

3. **Store Keystore Securely**:
   - ‚ö†Ô∏è **CRITICAL**: Back up keystore file (you cannot recover it)
   - Store keystore password in secure password manager
   - Never commit keystore to version control

## Phase 4: Screenshot Generation

### Step 4.1: App Store Screenshot Requirements (2025)

**Apple App Store (iPhone):**
- **Required Size**: 6.9-inch (iPhone 16 Pro Max) - 1320 x 2868 pixels
- **Optional Sizes**: 6.5-inch (older Pro Max), 5.5-inch (older iPhones)
- **Format**: PNG or JPEG
- **Quantity**: 1-10 screenshots per device size
- **Content**: Must be actual app screenshots (no mockups or external photos)

**Apple App Store (iPad):**
- **Required Size**: 13-inch iPad Pro - 2048 x 2732 pixels
- **Format**: PNG or JPEG
- **Quantity**: 1-10 screenshots
- **Note**: Only one iPad size required (automatically scales)

**Google Play Store (Android):**
- **Phone Screenshots**: At least 2, up to 8 screenshots
  - Min: 320px, Max: 3840px
  - Aspect ratio between 16:9 and 9:16
- **Tablet Screenshots** (Optional): 7-inch and 10-inch
  - Min: 1024px, Max: 3840px
- **Format**: PNG or JPEG (no alpha channel)

## Troubleshooting

### Common EAS Build Issues

**Issue: Build fails with "Unable to resolve module"**

**Cause**: Missing dependencies or incorrect Metro configuration

**Solution**:
```bash
cd /home/user/NestSyncV1.2/NestSync-frontend

# Clear Metro cache
npx expo start --clear

# Reinstall dependencies
rm -rf node_modules package-lock.json
npm install

# Retry build
eas build --platform ios --profile production
```

**Issue: iOS build fails with "No valid code signing identity found"**

**Cause**: Missing or expired certificates/provisioning profiles

**Solution**:
```bash
# Use EAS managed credentials (easiest)
eas credentials --platform ios

# Choose "Configure credentials"
# Select "Use Expo's managed credentials"
# Retry build
```

**Issue: Android build fails with "Duplicate resources"**

**Cause**: Conflicting dependencies or resource files

**Solution**:
```bash
# Check for duplicate dependencies
npm ls | grep duplicate

# Clean Android build cache
cd android && ./gradlew clean
cd ..

# Retry build
eas build --platform android --profile production
```

**Issue: Build succeeds but app crashes on launch**

**Cause**: Environment variables not set or incorrect

**Solution**:
```bash
# Verify all secrets are set
eas secret:list

# Check build logs for missing environment variables
# Add missing secrets:
eas secret:create --scope project --name MISSING_VAR --value "value"

# Rebuild with updated secrets
eas build --platform ios --profile production
```

### Common Store Submission Issues

**Issue: App Store rejection - "Guideline 2.1 - Performance - App Completeness"**

**Cause**: App crashes or major features non-functional during review

**Solution**:
- Test on real devices (not just simulator)
- Provide detailed review notes with test credentials
- Ensure backend is stable and not rate-limited
- Fix all crash-on-launch issues

**Issue: App Store rejection - "Guideline 5.1.1 - Privacy - Data Collection and Storage"**

**Cause**: Missing or incomplete privacy policy

**Solution**:
- Ensure privacy policy URL is accessible and comprehensive
- Explicitly mention Canadian data residency and PIPEDA compliance
- Update data collection declarations in App Store Connect
- Add in-app privacy information screen (optional but recommended)

**Issue: Google Play rejection - "Policy violation: User data"**

**Cause**: Data safety section incomplete or misleading

**Solution**:
- Review and complete all data safety questions
- Explicitly declare health data collection
- Mention encryption and secure storage
- Provide data deletion instructions

**Issue: TestFlight build not appearing for testers**

**Cause**: Export compliance not configured or build processing incomplete

**Solution**:
- Wait 5-30 minutes for build processing
- Check export compliance settings in App Store Connect
- Verify `ITSAppUsesNonExemptEncryption: false` in app.json
- Re-upload build if processing fails

### Backend Connectivity Issues

**Issue: App shows "Unable to connect to server" after deployment**

**Cause**: Production GraphQL endpoint misconfigured or backend not responding

**Solution**:
```bash
# Test backend health
curl https://nestsync-api.railway.app/health

# Test GraphQL endpoint
curl https://nestsync-api.railway.app/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{ __schema { types { name } } }"}'

# If backend is down, check Railway logs:
railway logs --project nestsync-backend

# Verify environment variables in Railway dashboard
railway variables list
```

**Issue: Authentication fails with "Invalid JWT signature"**

**Cause**: JWT secrets mismatch between frontend and backend

**Solution**:
```bash
# Verify production secrets match
# Backend (Railway dashboard): JWT_SECRET
# Frontend (EAS secrets): EXPO_PUBLIC_JWT_SECRET (if exposed) or backend handles

# Ensure backend JWT secret is set in Railway:
railway variables set JWT_SECRET="your-production-secret"

# Redeploy backend to apply changes
railway up
```

**Issue: WebSocket connections fail in production**

**Cause**: Unencrypted `ws://` used instead of `wss://`

**Solution**:
- Verify `lib/graphql/client.ts` uses `getWebSocketUrl()` function
- Ensure production endpoint starts with `https://` (auto-converts to `wss://`)
- Check CORS settings allow WebSocket connections
- Review Railway logs for WebSocket handshake errors
